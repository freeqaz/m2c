# VMX128 Support

m2c supports VMX128 SIMD instructions used by the Xbox 360 Xenon CPU. This document covers the supported instructions, calling conventions, and usage with decompilation workflows.

## Overview

VMX128 is an extension of PowerPC AltiVec (VMX) used exclusively by the Xbox 360's Xenon processor. Key differences from standard AltiVec:

- **128 vector registers** (v0-v127) instead of 32
- **Extended instruction encodings** using VX128 formats
- **Additional instructions** for graphics and physics operations
- **Different calling convention** for the extended registers

## Calling Convention

The Xbox 360 uses a specific calling convention for VMX128 registers:

| Registers | Usage | Preserved |
|-----------|-------|-----------|
| v0 | Return value | No |
| v1-v13 | Function arguments | No |
| v14-v19 | Temporary (caller-saved) | No |
| v20-v31 | Saved (callee-saved) | Yes |
| v32-v127 | Extended temporaries | No |

## Usage with dc3-decomp

When using m2c with Xbox 360 assembly from the [dc3-decomp](https://github.com/milohax/dc3-decomp) project:

```bash
# List functions in an assembly file
python3 tools/asm_to_m2c.py build/373307D9/asm/xdk/nuiapi/nuiskeleton.s -l

# Decompile a specific function
python3 tools/asm_to_m2c.py build/373307D9/asm/xdk/nuiapi/nuiskeleton.s \
  -f XMMatrixRotationAxis | python3 ~/code/milohax/m2c/m2c.py -t ppc -
```

The `asm_to_m2c.py` tool handles format conversion from dtk disassembly output to m2c-compatible assembly, including normalization of VMX128 instruction arguments.

## Example Output

Input assembly:
```asm
vmsum3fp128 v63, v1, v1
vrsqrtefp128 v0, v63
vmulfp128 v0, v1, v0
vsel128 v127, v0, v11, v127
```

Decompiled C:
```c
temp_v63 = __vmsum3fp128(v1, v1);      // 3D dot product (magnitude squared)
temp_v0 = __vrsqrtefp128(temp_v63);    // 1/sqrt estimate
temp_v0 = v1 * temp_v0;                // normalize
v127 = __vsel128(temp_v0, v11, v127);  // conditional select
```

## Technical Notes

### Instruction Encoding

VMX128 instructions use several encoding formats:
- **VX128**: Basic 3-operand format
- **VX128_1**: Load/store with indexed addressing
- **VX128_2**: Two-operand with immediate
- **VX128_3**: Three-operand with immediate
- **VX128_4**: Rotate/mask insert operations
- **VX128_5**: Five-bit immediate operations
- **VX128_P**: Permute with 7-bit immediate
- **VX128_R**: Rotate operations

### Comparison Results

VMX128 comparisons produce per-lane masks (`0xFFFFFFFF` or `0x00000000`), not simple booleans. This affects subsequent `vsel128` operations which use the mask for bitwise selection.

### Standard AltiVec Compatibility

m2c also supports standard AltiVec instructions (without the `128` suffix) that may appear alongside VMX128 code:
- `vmaddfp`, `vnmsubfp` - Multiply-add operations
- `vsel` - Vector select
- `vmr` - Vector move register
